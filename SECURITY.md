# Security Policy

This repository keeps security docs intentionally clean for initial publication.

Canonical security docs:
- `docs/SECURITY.md`
- `docs/SECURITY_IMPROVEMENTS.md`

Security issue records are intentionally not pre-populated.
Use private reporting as described in `docs/SECURITY.md`.
