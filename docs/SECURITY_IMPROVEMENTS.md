## Design Strategy For Future Population

Use this workflow for every new security item:
1. Identify: source (report, scan, test), scope, affected versions.
2. Classify: severity, exploitability, and business impact.
3. Assign: owner and target release.
4. Mitigate: temporary guardrail if immediate fix is not possible.
5. Fix: code/config/docs changes.
6. Validate: automated tests + manual verification evidence.
7. Release: ship fix and link release artifact/reference.
8. Review: capture lessons learned and follow-up hardening tasks.
