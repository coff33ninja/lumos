# Transparency

This project maintains transparency through:

- public release notes and compatibility details
- explicit security policy and hardening backlog
- known issues/limitations tracking
- roadmap and future-scope documentation

## Where To Look

- `docs/RELEASE_NOTES.md`
- `docs/VERSIONING.md`
- `docs/SECURITY.md`
- `docs/SECURITY_IMPROVEMENTS.md`
- `docs/KNOWN_ISSUES_AND_LIMITATIONS.md`
- `docs/FUTURE.md`

## Operating Principle

The goal is practical, user-controlled software with documented behavior, known limits, and auditable release changes.
